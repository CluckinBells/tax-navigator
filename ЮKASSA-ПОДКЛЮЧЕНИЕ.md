# Подключение оплаты ЮKassa в бота — пошагово

Оплата картой/SberPay/ЮMoney прямо в Telegram, с чеком по 54-ФЗ. Код бота уже готов
(`bot/server.js`) — осталось настроить токены и запустить сервер.

---

## ЭТАП А. Привязать ЮKassa к боту (в ЛК ЮKassa + @BotFather)

### Шаг 1. Магазин ЮKassa «для Telegram-бота»
⚠️ **Важно:** магазин должен быть зарегистрирован именно для приёма платежей в боте,
а НЕ для сайта. Если при заявке выбрали «сайт» — в ЛК ЮKassa добавьте отдельный
способ подключения «Telegram-бот» (или напишите менеджеру ЮKassa).
Запишите свой **shopId**.

### Шаг 2. Получить платёжный токен в @BotFather
1. Откройте **@BotFather** → `/mybots` → выберите **@taxes_navigator_bot**
2. **Bot Settings** → **Payments**
3. В списке провайдеров выберите **ЮKassa (YooKassa)**
4. BotFather проведёт через авторизацию в ЛК ЮKassa и привязку магазина
   (нужна роль «Владелец» или «Управляющий» в ЮKassa)
5. Получите **платёжный токен** — длинная строка вида `390540012:LIVE:xxxxx`
   - есть **TEST**-токен (для проверки, деньги не списываются)
   - и **LIVE**-токен (реальные платежи)
6. **Сохраните оба токена.**

> 💡 Сначала всё проверьте на TEST-токене (карта 1111 1111 1111 1026, любой срок, CVC 000),
> и только потом ставьте LIVE.

## ЭТАП Б. Запустить сервер бота на Amvera

Выбран **Amvera** — российский хостинг (рублёвая оплата), но с заграничным регионом
(чтобы бот не лагал с Telegram в 2026). Деплой через `git push`. Конфиг `amvera.yml` уже готов.

### Шаг 3. Создать проект на Amvera
1. Зарегистрируйтесь на **https://amvera.ru** (дают стартовый баланс на пару недель).
2. Создайте новый проект → тип **Node.js** → **сервер**.
3. ⚠️ Выберите **заграничный регион** (Варшава) — иначе бот может тормозить с Telegram.

### Шаг 4. Загрузить код (git push)
Amvera даст адрес репозитория. Из папки проекта:
```powershell
cd "C:\Users\XDot PC\OneDrive\Desktop\tax-navigator"
git remote add amvera https://git.amvera.ru/ВАШ_ЛОГИН/ВАШ_ПРОЕКТ
git push amvera dev:master
```
*(пушим ветку `dev` — там код с ЮKassa. Когда применим в `main`, будете пушить `main:master`.)*

### Шаг 5. Задать секреты в панели Amvera
В проекте → раздел **Переменные/Секреты** добавьте (как **секреты**, не открытым текстом):
```
BOT_TOKEN       = токен_бота_из_BotFather
PROVIDER_TOKEN  = платёжный_токен_ЮKassa_из_шага_2
WEBAPP_URL      = https://cluckinbells.github.io/tax-navigator/webapp/index.html
PRO_PRICE_RUB   = 1990
VAT_CODE        = 1
DATA_DIR        = /data
```
- `VAT_CODE=1` — «без НДС» (для ИП на УСН/НПД). Другой режим — уточните код в ЛК ЮKassa.
- `DATA_DIR=/data` — чтобы список оплативших не терялся при перезапуске.
- Amvera выдаст вашему сервису HTTPS-адрес вида `https://ВАШ_ПРОЕКТ.amvera.io` — запишите его.

### Шаг 6. Поставить вебхук
Чтобы Telegram присылал боту события (оплаты, /start), откройте в браузере один раз:
```
https://api.telegram.org/botВАШ_BOT_TOKEN/setWebhook?url=https://ВАШ_ПРОЕКТ.amvera.io/webhook/ВАШ_BOT_TOKEN
```
Должно вернуть `{"ok":true,...}`.

## ЭТАП Г. Проверить

1. Откройте бота → калькулятор → нажмите Pro → «Оформить Pro».
2. На TEST-токене оплатите тестовой картой — Pro должен открыться автоматически,
   а в чат придёт «🎉 Pro активирован навсегда».
3. Всё работает → замените TEST-токен на LIVE (шаг 3) и перезапустите сервер.

## Как это работает (схема)

```
Клиент жмёт «Оформить Pro»
   → Mini App просит у сервера ссылку на оплату
   → сервер создаёт инвойс ЮKassa (рубли, чек 54-ФЗ)
   → клиент платит картой прямо в Telegram
   → Telegram присылает боту «оплачено» (successful_payment)
   → сервер выдаёт Pro навсегда + чек уходит на email клиента
```

## Частые ошибки (из доков ЮKassa)
- **«Вы не можете принимать платежи с карт»** → магазин не зарегистрирован для бота;
  напишите менеджеру ЮKassa, укажите shopId.
- **BOT_PRECHECKOUT_TIMEOUT** → сервер не ответил за 10 сек; проверьте, что сервер запущен
  и вебхук стоит правильно.
- **Оплата не идёт в веб-версии Telegram** → это норма, платежи работают только
  в мобильном/десктоп-приложении Telegram.
